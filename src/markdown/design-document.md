## Prompt:

You’re about to create a detailed design document for a web application. This document will help understand the needs of the project/user to generate high-quality code using the following tech stack:
	•	Frontend: React + Next.js + TypeScript + Tailwind CSS + Shadcn UI + Lucide Icons + Custom form components
	•	Backend: Next.js server actions (if using App Router), Prisma ORM
	•	Database: PostgreSQL

Please answer the following questions as clearly and thoroughly as possible:

⸻

### 💡 1. App Overview
	•	What is the name of your app?
	•	Describe what your app does in 2-3 sentences.
	•	Who is the target audience or user base?
	•	What specific problem(s) does it solve?

⸻

### 🧱 2. Core Features
	•	What are the main features of the app? List them in bullet points.
	•	Are there any secondary or nice-to-have features?
	•	Are there any user roles or permissions (e.g., admin, guest, editor)? What can each role do?

⸻

### 👤 3. User Journey
	•	Describe the typical user flow: from signing up to completing their main task in the app.
	•	Are there any onboarding steps or tutorials?

⸻

### 📊 4. Data Model & Entities
	•	What are the main entities or database tables? (e.g., User, Project, Task, etc.)
	•	What fields or attributes should each entity have?
	•	How do these entities relate to each other (e.g., one-to-many, many-to-many)?

⸻

### 🔐 5. Authentication & Authorization
	•	Will users sign up and log in?
	•	What kind of authentication do you need? (e.g., email/password, OAuth, magic links)
	•	Do different users have different access levels or capabilities?

⸻

### 🖼️ 6. UI/UX & Design
	•	Do you have any specific UI ideas or references (e.g., Figma link, existing apps you like)?
	•	Should the layout be mobile-first or responsive?
	•	Do you need any components like dashboards, tables, forms, modals, etc.?

⸻

### 🔔 7. Notifications, Emails, or Integrations
	•	Should the app send email notifications? If yes, when and for what?
	•	Will it integrate with other tools or APIs (e.g., Stripe, Slack, Google Calendar)?
	•	Are there any background jobs or cron-like tasks?

⸻

### 📈 8. Scalability & Performance
	•	How many users do you expect initially and long-term?
	•	Are there any performance-critical parts of the app?

⸻

### 🚀 9. Admin or Backoffice Features
	•	Do you need an admin dashboard?
	•	Should admins be able to manage users, data, or app settings?

⸻

### 🧪 10. Testing & Deployment
	•	Do you require test coverage for certain features (e.g., unit tests, e2e)?
	•	Do you have a preferred deployment platform (e.g., Vercel, Railway, Fly.io)?

⸻

### ✨ 11. Anything Else?
	•	Is there any additional context, idea, or constraint the AI should know before generating the design or code?

⸻

Once completed, this questionnaire will be turned into a structured design document and used to scaffold out the application's architecture, database, UI, and backend logic.

⸻
